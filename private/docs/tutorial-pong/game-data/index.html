Game Data

Now that we have a basic structure for our game laid out, we need to think about what kind of data exists in Pong.

The Building Blocks#

To do so, let's list out the objects in the game:

  • The ball
  • The player's paddle
  • The AI paddle

Now, let's dive a little deeper and think about the attributes each of these items has.

The Ball#

  • An (x, y) position
  • An x velocity
  • A y velocity

The Player's Paddle#

  • An (x, y) position
  • A size
  • A score

The AI Paddle#

  • An (x, y) position
  • A size
  • A score
note

The AI Paddle may have more data associated with it depending on how sophisticated the AI controlling the paddle is. In this tutorial, we'll make a fairly 'dumb' AI to control the paddle. As a follow up to the guide, try to get more creative with the behavior of the AI!

Data to Code#

Defining Structures#

With these attributes identified, we can go ahead and translate this data into code that our game can run off of.

We'll use the same general principles we identified in the Snake tutorial here.

Go ahead and put the following code above void setup()

pong.ino
struct Paddle { // Handles both the AI paddle and player paddle    int x;    int y;    int size;    int score;};
struct Ball {    float x;    float y;    float xVel;    float yVel;};

This defines the blueprints for data packages representing the platform and the tower, following the elements we identified earlier.

tip

We use float to represent the ball's position and velocity since we need decimal point accuracy to allow for a large variation of ball movement vectors at reasonable speeds given our tick rate. Imagine if we were updating the ball's position by 5 units at a time!

Initializing objects#

Now that we've defined how our data is packaged, we can 'initialize' our object like so:

pong.ino
... // Struct code
Paddle playerPaddle = {0};Paddle aiPaddle = {0};Ball ball = {0};

The platform now lives in the computer's memory and will store value that we give to them according to its struct blueprint.

Finally, we just need to set the data for these objects to appropriate values at the start of our game. To make this a little easier on us in the future though, we can notice that there are two cases where we need to reset game data:

  • When a new game starts
  • When a new round starts

So, we can to identify the data that needs to get reset at each stage. In particular, we can observe that every time a new game starts, a new round also starts. So, starting a new round is a special case of starting a new game where we reset the ball's position and velocity.

New Round#

Let's first write a function to reset round data:

pong.ino
void newRound() {    ball.x = WIDTH / 2; // Set the ball's column to be in the middle    ball.y = HEIGHT / 2; // Set the ball's row to be in the middle    ball.xVel = ((float) (rand() % 21) - 10.0) / 10.0; // Pick a random x velocity for the ball between (-1, 1)    while (ball.xVel < 0.2 && ball.xVel > -0.2) { // While the x velocity is below a magnitude of 0.2, pick a new velocity        ball.xVel = ((float) (rand() % 21) - 10.0) / 10.0; // Pick a random x velocity for the ball between (-1, 1)    }    ball.yVel = ball.xVel = ((float) (rand() % 21) - 10.0) / 10.0; // Pick a random y velocity for the ball between (-1, 1)    while (ball.yVel < 0.1 && ball.yVel > -0.1) { // While the y velocity is below a magnitude of 0.1, pick a new velocity        ball.yVel = ball.xVel = ((float) (rand() % 21) - 10.0) / 10.0; // Pick a random y velocity for the ball between (-1, 1)    }}

This code resets the ball to be in the middle of the LED grid, and then assigns a random velocity to the ball within a certain bound. We restrict the velocity to a certain bound because we don't want the ball to travel too slowly, or else the game would take way too long and be too easy (or impossible to finish if the velocity is 0!).

New Game#

Now that we have the data reset procedure for a new round defined, we can identify what we need to reset for a new game, and then use our new round function to reduce the amount of code we need to duplicate:

pong.ino
void initGame() {    playerPaddle.score = 0; // Set player score to 0    playerPaddle.size = 4; // Set player paddle size to 4 blocks across    playerPaddle.x = 1; // Set position of player paddle horizontally    playerPaddle.y = HEIGHT / 2 - playerPaddle.size / 2; // Center player paddle vertically    aiPaddle.score = 0; // Set AI score to 0    aiPaddle.size = 4; // Set AI paddle size to 4 blocks across    aiPaddle.x = WIDTH - 2; // Set position of AI paddle horizontally    aiPaddle.y = HEIGHT / 2 - playerPaddle.size / 2; // Center AI paddle vertically    newRound(); // Call our new round function, handling round reset logic}

As we see, reseting a game is the same as initializing a new game. We set scores to zero, paddle positions and sizes, and then start a new round.

Constants#

We'll also defined a constant here that will represent the number of points a player needs to score to have won the game. We can put this constant variable code right below our initialized objects:

pong.ino
... // Object code
const int WINNING_SCORE = 6;

In this case, a player needs to score six points to win the game of Pong.
