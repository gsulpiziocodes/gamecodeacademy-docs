Game Update

With loop structure and data structures defined for the game, we now have to use our loop's structure to update our data, bringing one game state to the next.

Update Structure#

Here's a general outline for the structure of our update function:

  • Get user input
  • Decide which way the Snake should move
  • Check if the Snake is going to hit anything that will affect the game state & handle change
  • Move the Snake

User Input#

First thing's first, users interact with the Snake game through the joystick. Without the user input, nothing interesting happens in the game.

So, let's begin working on void updateSnake() by writing code to determine the joystick's position during a game tick:

snake.ino 
void updateSnake() {  // Read joystick values  int xValue = analogRead(joyX);  int yValue = analogRead(joyY);  int xMap = map(xValue, 0, 1023, -10, 10);  int yMap = map(yValue, 0, 1023, -10, 10);}

The above code reads in the X and Y values from the joystick, and then maps the raw analog signal range (0, 4095) to a more reasonable (-10, 10) range in the variables xMap and yMap.

With the joystick values read in, we can process them to decide which direction the user wants the Snake to turn:

snake.ino 
void updateSnake() {  ... // EXISTING CODE
  // Handle movement logic  if (abs(xMap) > 3 || abs(yMap) > 3) { // Remove noise by making sure the joystick is turned at least 30% off an axis      if (abs(xMap) > abs(yMap)) {          if (xMap > 0) {              snake.dir = Direction::right;          } else {              snake.dir = Direction::left;          }      } else {          if (yMap > 0) {              snake.dir = Direction::down;          } else {              snake.dir = Direction::up;          }      }  }
info

The abs(val) function returns the absolute value of the number supplied to it.

Collision Detection#

A core aspect of all game development is collision detection. This is the process of determining when game objects are intersecting with each other in the game world, and then taking appropriate action depending on the situation.

Out of Bounds#

The most common form of collision detection in retro-game is arguably out-of-bounds cases. These cases are situations where game objects would travel outside of the game grid if interventions weren't made. This is especially dangerous in a language such as C (which Arduino is programmed in), since the game world is represented as an array, and referencing outside the array can lead to dangerous memory exceptions.

In Snake, we just need to make sure that the Snake will not move outside the game bounds of the 16x16 grid. If it does, the player has lost the game, and we should reset the player's progress.

We can accomplish this conditional check and appropriate action with the following code:

tip

Place the following above updateSnake() in your existing code

snake.ino 

// Function to move the position (pos) in the direction (dir) supplied// Returns the value of the new position.Pos updatePos(Pos pos, Direction dir) {    switch (dir) {        case Direction::up: {            pos.y++;            return pos;        }        case Direction::down: {            pos.y--;            return pos;        }        case Direction::left: {            pos.x--;            return pos;        }        case Direction::right: {            pos.x++;            return pos;        }    }}
// Function to check if the position is in a valid location in the game worldbool isValidPosition(Pos pos) {    if (pos.x < 0 || pos.x >= WIDTH || pos.y < 0 || pos.y >= HEIGHT) { // Is position out of bounds?        return false;    }    return true;}
info

The ! in front of isValidPosition(newPos) makes true go to false and false go to true.

Finally, let's update our updateSnake function to call the updatePos function.

snake.ino 
void updateSnake() {  ... // EXISTING CODE
  // Check collision  Pos newPos = updatePos(snake.positions[0], snake.dir);  if (!isValidPosition(newPos)) {      initGame();      return;  }}

Snake Hits snake#

A cornerstone of Snake is that running into yourself loses you the game. To implement this behavior in our game, we can just make a quick modification to our bool isValidPosition(Pos) function:

snake.ino 
// Function to check if the position is in a valid location in the game worldbool isValidPosition(Pos pos) {    if (pos.x < 0 || pos.x >= WIDTH || pos.y < 0 || pos.y >= HEIGHT) { // Is position out of bounds?        return false;    }    for (int i = 0; i < snake.len; i++) { // Loop through all of the snake pieces and check if the new position intersects any of them        if (pos.x == snake.positions[i].x && pos.y == snake.positions[i].y) {            return false;        }    }    return true;}

Snake Hits Food#

When the Snake hits a piece of food, we need to add to its tail length and reposition the food in the game world. Since the food is a single block in the game world, detecting a collision is rather straightforward. In this case, the complex part is deciding what it looks like to add a new block to the Snake's tail.

In our implementation it actually turns out to be rather straightforward to make the Snake longer - we'll cover it in detail next. For now, we'll just increment the length by one and make it work later:

snake.ino 
void updateSnake() {  ... // EXISTING CODE
  // Check food  bool didEat = newPos.x == food.pos.x && newPos.y == food.pos.y;  if (didEat) { // Move food to new random location      food.pos.x = rand() % WIDTH;      food.pos.y = rand() % HEIGHT;      snake.len++;  }}

Moving the Snake#

Now that we've handled appropriate collision detection events, we need to update the game state by actually moving the Snake to the new location. Thinking about the snake as a collection of squares, what we want to do is add a new square where the new position is (the snake's head), and then remove the last square on the snake's tail. This will effectively give the illusion of our Snake traveling.

But what about the condition of making our snake grow longer? In fact, the only thing we need to do differently from the procedure outlined above is not remove the last square of the Snake's tail - simply add a new square to become the Snake's head and the Snake will grow by one!

Let's take a look at an implementation of this logic:

snake.ino 
void updateSnake() {  ... // EXISTING CODE
  // Move Snake  Pos newPositions[256]; // Make a new temporary array in memory that we can copy Snake data into  newPositions[0] = newPos; // Head of Snake becomes new position  for (int i = 1; i < snake.len; i++) { // Shift the rest of the Snake's square positions back by one      newPositions[i] = snake.positions[i - 1]; // If snake.len stays the same, the last square is not copied over. If snake.len was incremented in collision detection, it is.  }  memcpy(snake.positions, newPositions, sizeof snake.positions); // Copy the the contents of our temporary snake array (newPositions) into the actual gamedata array (snake.positions)}

With that, our Snake's position updates correctly every game tick!

One Last Edge Case#

There's one more thing we have to consider before we move on from our game update implementation. Currently, when the Snake eats the food we just move it to a new random position on the screen. But what if the food is moved on top of the Snake? This is not something we want to allow to happen, so we can consider it an edge case to the food relocation algorithm.

To deal with this, we can implement a quick, but admittedly inefficient, fix. We can check if the food is spawned on the Snake - if so, move the food again - and repeat this process until the food is not on the snake:

snake.ino 
void updateSnake() {  ... // EXISTING CODE
  // Move food if it spawned on snake  while (!isValidPosition(food.pos)) {      food.pos.x = rand() % WIDTH;      food.pos.y = rand() % HEIGHT;  }}
