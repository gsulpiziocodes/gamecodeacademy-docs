Game Render

At this point, our game is actually fully functional, there's just one problem - we can't see any of it! In order to visualize the data structures and their updates, we need to draw them onto the LED panel using FastLED.

Let's restrict all of our rendering logic to the void renderGame() function we defined earlier. This is good encapsulation practice in C.

Here's the basic outline for how the render function will operate:

stackit.ino 
void renderGame() {    // Zero LED display    memset(leds, 0, sizeof leds); // Ensures that our LEDs are all set to black by default
    // Render tower    ...
    // Render platform    ...
    // Draw LEDS    FastLED.show(); // FastLED library call to draw the LEDs}

You should add that code snippet above void setup().

note

The general structure of rendering pixel games is quite similar in between games, so you may notice quite a bit of overlap between rendering Snake, Stack It, and Pong.

Translation of Matrix#

The indexing of the LED grid is not the same as the indexing of our game grid out of the box. Thus, if we try to map our game indexes to the pixel grid in a 1-1 manner, it won't really look like anything coherent.

To get the indexes of the game grid and the LED grid to match up, we can define a new function int translateMatrixIndex(int i). This function will take in a game matrix index and translate it to its corresponding LED matrix index. It's not important to understand the details of this translation, just that it is important to ensure the indexes between the two matrices in row-column format match up:

stackit.ino 
int translateMatrixIndex(int i) {    int r = i / WIDTH;    int c = i % WIDTH;    i = c * WIDTH + (15 - r);    if (i / 16 % 2 == 1)        return 2 * (7.5 - (i % 16)) + i;    return i;}

You should add that code snippet above void renderGame().

Render the Tower#

Let's first draw the tower. To do so, we just need to fill in the LEDs red where the platform currently exists, and black otherwise. So, we can just loop through the entire 16x16 game grid stored in the array tower, check if that index is true (corresponding to a tower block), and then set that LED to red using hex values.

stackit.ino 
void renderGame() {    ...
    // Render tower    for (int i = 0; i < NUM_LEDS; i++) { // Loop through the tower array        if (tower[i]) { // Check if there is a tower block at this index            leds[translateMatrixIndex(i)] = 0xff0000; // Set LED at this index to red        }    }
    ...}

Render the Platform#

Now let's render the platform. To do so, we can simply loop over the size of the platform, and draw the corresponding block over each iteration. We'll calculate the appropriate index using the row-column to index formula, offset by the current value of the size iterator i. We'll also make the platform green using hex values.

stackit.ino 
void renderGame() {    ...
    // Render platform    for (int i = 0; i < platform.size; i++) { // Loop through the size of the platform with iterator i        int index = platform.layer * WIDTH + (platform.x + i); // Use row-column to index formula and size iterator to calculate index of platform block        leds[translateMatrixIndex(index)] = 0x00ff00; // Set LED at index to green    }
    ...}

Render the Blinking Screen#

Earlier we referenced a function called void renderBlink(CRGB color), which was called upon reaching the winning and losing game state. The purpose of this function is to make every LED on the screen blink the color of the color variable (in hex).

In particular, we will have four blink cycles (implemented through a for loop) where the screen will go from color to black. We'll accomplish the flashing through simply rewriting every element in the LED array to be the desired color, calling the FastLED draw function, and then delaying the microcontroller by 250 milliseconds.

Go ahead and paste the following implementation of void renderBlink(CRGB color) above void updateGame():

stackit.ino 
void renderBlink(CRGB color) {    for (int i = 0; i < 4; i++) { // Loop through four blink cycles        for (int j = 0; j < NUM_LEDS; j++) { // Set all LEDs to 'color'            leds[j] = color;        }        FastLED.show(); // Draw LEDs        delay(250); // Wait 250ms        for (int j = 0; j < NUM_LEDS; j++) { // Set all LEDs to black            leds[j] = 0x000000;        }        FastLED.show(); // Draw LEDs        delay(250); // Wait 250ms    }}
