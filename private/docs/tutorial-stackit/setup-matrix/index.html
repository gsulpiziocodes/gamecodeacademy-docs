Stack It Setup

At this point we're ready to begin writing some code. Go ahead and open up your Ardiono IDE that you downloaded in the Getting Started section and make sure that your microcontroller is connected to your computer.

Barebones Code#

We'll go ahead and name our code file stackit.ino, and will refer to it as such through this tutorial. You can name it whatever you want.

You can go ahead and copy and paste the following code block into your file. It just sets up some basic constants and structure which will be useful to us while programming.

stackit.ino 
#define WIDTH 16#define HEIGHT 16#define NUM_LEDS WIDTH * HEIGHT
#define DATA_PIN 0#define CLICK_PIN 1#define joyX A0#define joyY A1
void setup() {    pinMode(CLICK_PIN, INPUT_PULLUP); // Set joystick button pin mode as input pullup}
void loop() {
}

Setup FastLED Library#

FastLED is a library that will allow us to draw our game on the 16x16 pixel matrix. We interact with the library through "objects" that it provides, which are just packages of data that have names associated with them. Let's import the library and create one of its object.

At the top of your file add:

stackit.ino 
#include <FastLED.h>

This adds the library to your project.

Now we can instantiate the FastLED object in our code and configure it in the setup function like so:

stackit.ino 

CRGB leds[NUM_LEDS] = {0}; // Instantiate the FastLED Object
void setup() {    pinMode(CLICK_PIN, INPUT_PULLUP); // Set joystick button pin mode as input pullup    FastLED.setBrightness(20);    FastLED.addLeds<WS2812B, DATA_PIN, GRB>(leds, NUM_LEDS); // Configure the object}

FastLED is all ready to go!

General Game Structure#

Our game plays through the loop, so we can think about what kind of operations need to happen every loop for our game to work.

In particular, we need to:

  • Update the game state
  • Render the game to the LED display
  • Delay till next game tick

To accomplish the first two steps, we can define two functions, updateGame() and renderGame().

The final step is supported through Arduino already using the delay(ms) function, which delays the microcontroller's next process in milliseconds.

The implementation of this structure idea looks like this:

stackit.ino 

void updateGame() {
}
void renderGame() {
}
void loop() {  // Update game  updateGame();
  // Render Game to LEDs  renderGame();
  // Wait for next tick  delay(???);}
info

Delay here is set to ??? because the tick rate in Stack It gets faster the higher the layer becomes. We'll fix this later.
