Game Render

At this point, our game is actually fully functional, there's just one problem - we can't see any of it! In order to visualize the data structures and their updates, we need to draw them onto the LED panel using FastLED.

Let's restrict all of our rendering logic to the void renderGame() function we defined earlier. This is good encapsulation practice in C.

Here's the basic outline for how the render function will operate:

pong.ino
void renderGame() {  // Zero display  memset(leds, 0, sizeof leds);
  // Render middle line
  // Render player paddle
  // Render AI paddle
  // Render ball
  // Render 4-Digit score
  // Render colon
  // Draw LEDs  FastLED.show();}

You should add that code snippet above void setup().

note

The general structure of rendering pixel games is quite similar in between games, so you may notice quite a bit of overlap between rendering Snake, Stack It, and Pong.

Translation of Matrix#

The indexing of the LED grid is not the same as the indexing of our game grid out of the box. Thus, if we try to map our game indexes to the pixel grid in a 1-1 manner, it won't really look like anything coherent.

To get the indexes of the game grid and the LED grid to match up, we can define a new function int translateMatrixIndex(int i). This function will take in a game matrix index and translate it to its corresponding LED matrix index. It's not important to understand the details of this translation, just that it is ensures the indexes between the two matrices in row-column format match up:

pong.ino
int translateMatrixIndex(int i) {    int r = i / WIDTH;    int c = i % WIDTH;    i = c * WIDTH + (15 - r);    if (i / 16 % 2 == 1)        return 2 * (7.5 - (i % 16)) + i;    return i;}

You should add that code snippet above void renderGame().

Render the Middle Line#

In pong, there is a middle line separating the two ends of the arena between the player's paddles. To render a middle line, we'll use a for loop to draw alternating green squares on columns 7 and 8:

pong.ino
void renderGame() {  ...
  // Render middle line  for (int i = 0; i < HEIGHT; i+=2) { // iterate by i+=2 each time since squares will be alternating (we skip every other index)      int indexLeft = i * WIDTH + 7; // Get index for column 7      int indexRight = i * WIDTH + 8; // Get index for column 8      leds[translateMatrixIndex(indexLeft)] = 0x00ff00;      leds[translateMatrixIndex(indexRight)] = 0x00ff00;  }}

Render the Paddles#

Now we can render the player's paddle and the AI paddle. The rendering procedure for both is basically the same: loop through the size of the paddle, use the row-column index formula with the iterator to find the index of the paddle square, and draw it red.

pong.ino
void renderGame() {  ...
  // Render player paddle  for (int i = 0; i < playerPaddle.size; i++) {      int index = (playerPaddle.y + i) * WIDTH + playerPaddle.x;      leds[translateMatrixIndex(index)] = 0xff0000;  }
  // Render AI paddle  for (int i = 0; i < aiPaddle.size; i++) {      int index = (aiPaddle.y + i) * WIDTH + aiPaddle.x;      leds[translateMatrixIndex(index)] = 0xff0000;  }}

Render the Ball#

As with other one-square objects, rendering the ball is quite straightforward:

pong.ino
void renderGame() {  ...
  // Render ball    int ballIndex = round(ball.y) * WIDTH + round(ball.x); // Round x and y position since it's a float and our game grid is indexed through integers    leds[translateMatrixIndex(ballIndex)] = 0x0000ff; // Set ball to blue}

Render 4-Digit Score#

Here's something new: rendering content onto the TM1637 display. For Pong, we'll have the player's 2-digit score show up on the left side of the display, and the AI's 2-digit score show up on the right side of the display. We will separate them by a colon.

The TM1637 library takes in a number to display on the screen. Thus, to properly conform to how we want to display the scores, we can create a variable fourScore which will hold the player's score in the thousands and hundreds slots, and the ai's score in the tens and ones slots. So, if the player's score is xx and the AI's score is yy, fourScore = xxyy.

With this number stored in fourScore we can draw it to our TM1637 display using a method call on our tm object defined in Pong Setup

pong.ino
void renderGame() {  ...
  // Render 4-Digit score  tm.setBrightness(0xff); // Set brightness of TM1637  int fourScore = playerPaddle.score * 100 + aiPaddle.score; // Calculate fourScore value  tm.showNumberDec(fourScore, true); // Show the value of fourScore on the TM1637 display}

Draw the Colon#

Drawing the colon turns out to be a little tricky, and it's not really that important to understand how it's done for almost every general case, so we'll just copy the code into our render method without worrying too much about it:

pong.ino
void renderGame() {  ...
  // Render colon  uint8_t segto = 0x80 | tm.encodeDigit((fourScore / 100)%10);  tm.setSegments(&segto, 1, 1);}

Render the Blinking Screen#

Earlier we referenced a function called void renderBlink(CRGB color), which was called upon reaching the winning and losing game state. The purpose of this function is to make every LED on the screen blink the color of the color variable (in hex).

In particular, we will have four blink cycles (implemented through a for loop) where the screen will go from color to black. We'll accomplish the flashing through simply rewriting every element in the LED array to be the desired color, calling the FastLED draw function, and then delaying the microcontroller by 250 milliseconds.

Go ahead and paste the following implementation of void renderBlink(CRGB color) above void updateGame():

pong.ino
void renderBlink(CRGB color) {    for (int i = 0; i < 4; i++) { // Loop through four blink cycles        for (int j = 0; j < NUM_LEDS; j++) { // Set all LEDs to 'color'            leds[j] = color;        }        FastLED.show(); // Draw LEDs        delay(250); // Wait 250ms        for (int j = 0; j < NUM_LEDS; j++) { // Set all LEDs to black            leds[j] = 0x000000;        }        FastLED.show(); // Draw LEDs        delay(250); // Wait 250ms    }}
